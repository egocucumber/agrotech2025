# РУКОВОДСТВО ОПЕРАТОРА

## 1. НАЗНАЧЕНИЕ ПРОГРАММЫ

### 1.1. Сведения о назначении программы

Программа AgroVzor предназначена для автоматизации и мониторинга процесса трехзвенной уборки урожая зерновых культур. Система обеспечивает автоматизированное взаимодействие между комбайном, бункером-перегрузчиком и зерновозом, осуществляет учет и контроль перемещения зерна, анализирует потери урожая, мониторит состояние техники и эффективность работы механизаторов.

### 1.2. Информация для понимания функций программы

**Основные функции программы:**

1. **Управление организационной структурой:**
   - Регистрация и управление предприятиями
   - Учет сотрудников и их допусков к технике
   - Учет парка сельскохозяйственной техники

2. **Планирование и учет полевых работ:**
   - Регистрация работ на полях
   - Учет рабочих выездов механизаторов
   - Планирование состава команды для выполнения работ

3. **Мониторинг и аналитика:**
   - Отслеживание потерь зерна при уборке урожая
   - Мониторинг состояния техники (вибрация, топливо)
   - Оценка эффективности работы механизаторов
   - Автоматический подбор команды для полевых работ

4. **Визуализация и отчетность:**
   - Отображение данных на интерактивных картах
   - Построение графиков и диаграмм
   - Формирование отчетов для анализа

### 1.3. Информация для эксплуатации программы

Программа работает через веб-браузер, не требует установки дополнительного программного обеспечения на компьютер оператора. Все данные хранятся на сервере и доступны с любого устройства с доступом в интернет.

## 2. УСЛОВИЯ ВЫПОЛНЕНИЯ ПРОГРАММЫ

### 2.1. Минимальный состав аппаратурных средств

- Персональный компьютер, планшет или смартфон
- Процессор: современный процессор с поддержкой JavaScript
- Оперативная память: 512 MB
- Разрешение экрана: не менее 1024x768 пикселей

### 2.2. Минимальный состав программных средств

- Операционная система: Windows 10+, macOS 10.14+, Linux (любое современное распределение)
- Веб-браузер: Chrome 90+, Firefox 88+, Edge 90+, Safari 14+
- JavaScript должен быть включен в браузере
- Доступ в интернет: не менее 1 Мбит/с

### 2.3. Максимальный состав аппаратурных и программных средств

- Любые современные устройства с веб-браузером
- Любые современные операционные системы
- Любые современные веб-браузеры с поддержкой ES6+

## 3. ВЫПОЛНЕНИЕ ПРОГРАММЫ

### 3.1. Последовательность действий оператора

**3.1.1. Загрузка программы:**

1. Откройте веб-браузер на вашем компьютере
2. Введите в адресной строке URL системы (например: `http://agrovzor.example.com`)
3. Нажмите Enter
4. Дождитесь загрузки главной страницы

**3.1.2. Запуск программы:**

Программа запускается автоматически при открытии страницы. Никаких дополнительных действий не требуется.

**3.1.3. Выполнение программы:**

После загрузки главной страницы вы можете:
- Просматривать данные на дашборде
- Переходить к различным разделам через меню навигации
- Выполнять операции через формы и кнопки интерфейса

**3.1.4. Завершение программы:**

Для завершения работы просто закройте вкладку браузера или перейдите на другую страницу. Данные автоматически сохраняются на сервере.

### 3.2. Описание функций

**3.2.1. Главная страница (Дашборд):**

- Отображение ключевых показателей эффективности (KPI)
- Графики потерь зерна, уровня топлива, эффективности механизаторов
- Критические оповещения и инциденты
- Статусы систем мониторинга

**3.2.2. Управление предприятиями:**

- Просмотр списка предприятий
- Создание нового предприятия
- Редактирование данных предприятия
- Удаление предприятия

**3.2.3. Управление сотрудниками:**

- Просмотр списка сотрудников
- Создание нового сотрудника
- Редактирование данных сотрудника
- Управление допусками к технике

**3.2.4. Управление техникой:**

- Просмотр списка техники
- Регистрация новой единицы техники
- Редактирование данных техники
- Управление типами и моделями техники

**3.2.5. Работы на поле:**

- Просмотр списка работ
- Создание новой работы на поле
- Просмотр истории работ
- Управление рабочими выездами

**3.2.6. Аналитика:**

- Расчет потерь зерна
- Анализ вибрации техники
- Мониторинг топлива
- Оценка эффективности механизаторов
- Подбор команды для полевых работ

**3.2.7. Инциденты:**

- Просмотр списка инцидентов
- Фильтрация по типу и серьезности
- Просмотр деталей инцидента на карте

### 3.3. Формат и возможные варианты команд

**3.3.1. Навигация:**

- Клик по пунктам меню для перехода между разделами
- Использование кнопки "Назад" браузера для возврата
- Использование поиска для быстрого поиска данных

**3.3.2. Работа с формами:**

- Заполнение полей формы
- Нажатие кнопки "Сохранить" для сохранения данных
- Нажатие кнопки "Отмена" для отмены изменений
- Нажатие кнопки "Удалить" для удаления записи (с подтверждением)

**3.3.3. Работа с таблицами:**

- Сортировка по столбцам (клик по заголовку столбца)
- Фильтрация данных через поля поиска
- Просмотр деталей записи (клик по строке)
- Редактирование записи (кнопка "Редактировать")

**3.3.4. Работа с графиками:**

- Наведение курсора на точки графика для просмотра значений
- Масштабирование графика (колесико мыши)
- Переключение между различными метриками

### 3.4. Ответы программы на команды

**3.4.1. Успешные операции:**

- Отображение сообщения об успехе (зеленое уведомление)
- Обновление данных на странице
- Возврат к списку или предыдущей странице

**3.4.2. Ошибки валидации:**

- Отображение сообщения об ошибке (красное уведомление)
- Подсветка полей с ошибками
- Описание проблемы в сообщении

**3.4.3. Предупреждения:**

- Отображение предупреждения (желтое уведомление)
- Запрос подтверждения для критических операций

## 4. СООБЩЕНИЯ ОПЕРАТОРУ

### 4.1. Тексты сообщений

**4.1.1. Сообщения об успешных операциях:**

```
✓ Предприятие успешно создано
✓ Сотрудник успешно добавлен
✓ Данные успешно сохранены
```

**4.1.2. Сообщения об ошибках:**

```
✗ Ошибка: Не удалось сохранить данные
✗ Ошибка: Поле "Наименование" обязательно для заполнения
✗ Ошибка: Соединение с сервером потеряно
```

**4.1.3. Предупреждения:**

```
⚠ Внимание: Вы уверены, что хотите удалить эту запись?
⚠ Предупреждение: Обнаружена аномалия вибрации на технике КМБ-001
```

**4.1.4. Информационные сообщения:**

```
ℹ Информация: Данные обновлены
ℹ Информация: Синхронизация данных завершена
```

### 4.2. Описание содержания сообщений

**Сообщения об успешных операциях:**
- Подтверждают, что операция выполнена успешно
- Данные сохранены и доступны для использования

**Сообщения об ошибках:**
- Указывают на проблему при выполнении операции
- Содержат описание проблемы и рекомендации по устранению

**Предупреждения:**
- Информируют о потенциально опасных операциях
- Требуют подтверждения перед выполнением

**Информационные сообщения:**
- Информируют о текущем состоянии системы
- Не требуют действий от оператора

### 4.3. Соответствующие действия оператора

**4.3.1. При успешных операциях:**

- Продолжите работу в обычном режиме
- Проверьте, что данные отобразились корректно

**4.3.2. При ошибках валидации:**

1. Прочитайте сообщение об ошибке
2. Проверьте заполнение полей формы
3. Исправьте ошибки в данных
4. Повторите попытку сохранения

**4.3.3. При ошибках соединения:**

1. Проверьте подключение к интернету
2. Обновите страницу (F5 или Ctrl+R)
3. Если проблема сохраняется, обратитесь к системному администратору

**4.3.4. При предупреждениях:**

1. Прочитайте предупреждение
2. При необходимости подтвердите операцию
3. Для критических предупреждений (аномалии, кражи) немедленно примите меры

**4.3.5. Действия оператора в случае сбоя:**

1. Сохраните текущее состояние (если возможно)
2. Обновите страницу (F5 или Ctrl+R)
3. Если проблема сохраняется, закройте и откройте браузер заново
4. Если проблема критическая, обратитесь к системному администратору

**4.3.6. Возможности повторного запуска программы:**

- Программа работает непрерывно на сервере
- Для повторного доступа просто откройте URL в браузере
- Данные сохраняются автоматически, потери данных не происходит
